Intro

Web 1.0, Web 2.0, and Web 3.0 each represent a different epoch in the internet's lifespan.

Web 1.0

Web 1.0 is called the "Read-Only Web" by Tim Berner-Lee and other experts and was around from the internet's creation until 2005. The Web's content was largely static, with content created by web-masters with HTML knowledge. There were means of searching for information. However were not much ability to exchange information between a website and a regular end user (between the producer of information and its consumer)

Web 2.0

Web 2.0 was the next generation of the web. Web 2.0 was enabled by "relationship technologies" and "read-write technologies." Unlike the previous generation of the web, in Web 2.0, users could interact with web pages. This allowed for relationship technologies to emerge, these paved the way for collaborative web sites such as MySpace, Facebook, StackOverFlow, and manny more. Many of these platforms are still in use today!

Web 3.0

Web 3.0 differs from the two previous iterations of the Web in that Web 3.0 targets on linked data sets. Tim Berners-Lee coined the term the Semantic-Web to refer to Web 3.0. In Web 3.0, meaningful information can be found by software agents, by using mata data; this information can also be evaluated and delivered. Web 3.0 is also different then Web 2.0 and Web 1.0 in that Web 3.0 is ubiquitous, available anytime, anywhere, and through many channels and devices; individualized, int that it can be filtered by friends or a trusted network; and efficient, int that information can be found efficiently.

Use Cases


The following diagrams are UML Use Case diagrams that illustrate the attempts of two web users to find a date using the web


Finding a date in Web 1.0


User 1 can post an add to a website (such as craigslist). User 2 can only view the add an respond via a separate platform. After talking through the separate platform the two users can decide to meet. This use case exemplifies the read-only format of Web 1.0

Finding a date in Web 2.0


User 1 has a profile social network (such Facebook). User 2 can browse the social network, and if so chooses can send User 1 a message directly via the social network. After communicating via the social network, the two users can meet up. This use case exemplifies the peer to peer interaction typical in Web 2.0

Finding a date in Web 3.0


User 1 and User 2 both sign up to a dating service. The dating service engine uses metadata collected by social network to suggest matches between users. The users can then converse via its platform. The use of data by an engine that did not collect the data exemplifies transfer of data by the web and its use
